**문제 구조**:

```
[정상 케이스] peak +3% 이상 달성 후 2% 역행 청산:
진입 → +3% peak → -2% 역행 = 출구 +1% 수익

[문제 케이스] peak +1~1.5% 수준에서 2% 역행 청산:
```

**결론**: trail_drop 2.0% 기준이 peak% < 2.0% 케이스에서 항상 손실을 발생시킴. 수익 케이스는 peak가 3~7%까지 상승한 후 drop이므로 손실이 없음.

### 가설 1: SPLIT_2ND trail_drop을 peak% 연동형으로 변경

**현재 문제**: trail_drop 2.0% 고정 → peak < 2.0%이면 무조건 손실.

**제안 메커니즘 (손익비 보장 로직)**:

```
peak% < 1.5% 구간: SPLIT_2ND를 trail로 청산하지 않고 SL_WIDE로 대체
  → 오히려 손절로 전환 (최소 -2.8% → -2.8% 동일하나 "trail에 의한 역설적 손실" 방지)

peak% 1.5%~2.5% 구간: trail_drop = 1.0%로 축소
  → exit = peak - 1.0% = 최소 +0.5% 보장

peak% >= 2.5% 구간: trail_drop = 2.0% 유지
  → exit = peak - 2.0% = 최소 +0.5% 보장
```

**기대 효과**: 6건의 SPLIT_2ND 손실 중 BIRB/SOON/RAY/API3 4건이 개선 대상.
- BIRB(peak 1.89%): trail_drop 1.0% 적용 시 최소 +0.89% 수익
- SOON(04-23, peak 1.12%): trail_drop 1.0% 적용 시 최소 +0.12% 수익
- RAY(peak 0.66%): peak < 1.5%이므로 SL 전환 → 손실 그대로이나 "trail로 손실"이라는 구조 모순 해소
- API3(peak 1.33%): trail_drop 1.0% 적용 시 최소 +0.33% 수익

**예상 개선 PnL**: 약 +3,000~5,000원/건 × 4건 = +12,000~20,000원

**검증 명세**:
- 대상 마켓: BIRB, SOON, RAY, API3 (4/22~4/27 동일 기간)
- 시뮬레이션: peak% 구간별 trail_drop 차등 적용
- 기준: 전체 38건 중 SPLIT_2ND 손실 변화율, 총 PnL 변화

### 가설 2: SPLIT_1ST 발동 조건에 "순손실 발동 차단" 추가

**현재 문제**: SPLIT_1ST가 roi <= 0.00% (원가 이하) 상태에서 발동되는 케이스 존재.

- PIEVERSE: SPLIT_1ST roi=-0.08%

**이 경우의 문제점**:
1. SPLIT_1ST 발동 자체가 손실이거나 0
2. 이후 SPLIT_2ND도 armed 조건 없이 trail만으로 발동
3. 전체 손실이 더 커짐 (작은 1st 손실 + 큰 2nd 손실)

**제안**:
```
SPLIT_1ST 발동 조건에 최소 roi 하한선 추가:
  현재: peak >= armed_pnl_pct AND drop >= split_1st_trail_drop

```

**효과**: BIRB, SOON(04-23), PIEVERSE 등 손실 상태에서 1st 발동하는 케이스 차단.
→ 대신 가격이 더 회복할 때까지 홀딩 or SL_WIDE에 의한 손절.
→ SPLIT_1ST가 정말 "익절" 역할을 할 때만 발동.

**리스크**: 일부 대형 상승장에서 SPLIT_1ST를 놓칠 수 있음 (단, peak가 충분히 올라갔을 때만 발동하므로 대부분 문제없음).
