## 3. 익절 극대화 가설

### 가설 H1: SPLIT_2ND trail_drop 래더링 (1차 0.7% / 2차 1.5%)

#### 가설 내용
현재 2.0% 단일 임계로 2차 청산 → **래더 방식으로 전환**: peak+1.5% 이상 도달 시 trail_drop 0.7%, 그 이하 시 trail_drop 1.5%

#### 구체적 파라미터 제안
- peak < 2.0% : trail_drop 현행 2.0% 유지 (작은 움직임 → 손절 위험 크므로 여유)
- 2.0% <= peak < 4.0% : trail_drop 1.5%
- peak >= 4.0% : trail_drop 0.7%

#### 근거 (실 데이터 기반)
- peak 4%+ 거래: CHIP(6.74→2.25 = 4.50갭), SONIC(4.90→1.75 = 3.15갭), PENGU-27(5.11→2.92 = 2.19갭), KAT(7.31→4.57 = 2.74갭) 4건
- 이들의 평균 갭 3.15%p → trail을 0.7%로 조이면 최대 **1.3~2.4%p** 추가 회수 가능
- peak 2~4% 구간 8건 평균 갭 2.13%p → trail 1.5%로 조이면 평균 **0.63%p** 추가 회수 가능
- KAT 사례: peak 7.31%에서 trail 2.0% = 5.31%에서 청산, trail 0.7%로 조이면 6.61%에서 청산 → **약 1.3%p 추가, ~3,000 KRW 추가 수익 추정**

#### 리스크
- trail이 0.7%로 좁아지면 일시적 조정(0.8~1.0% 하락)에도 청산 → peak 후 재상승 구간 포기 가능성
- CHIP 사례: peak 190 → drop 2.11% = 184에 청산. trail 0.7%였다면 peak 190 → 189에 청산 → peak 직후 바로 상승했다면 손실 없음, 그러나 실제로 190 이후 175까지 급락함 → 이 사례에서는 trail 타이트닝이 유리

#### 백테스트 검증 방법
- 대상 마켓: 최근 90일 OP 스캐너 진입 이력 전체 (KRW 기준 상위 15개 마켓)
- 시뮬레이션 방식: BacktestSimulator에 trail_drop 래더 파라미터 추가, peak 구간별 3단계 적용
- 측정 지표: 17건 SPLIT_2ND_TRAIL 수익 거래의 2차 청산 가격 재계산, avg_roi% 비교
- 검증 기간: V129 배포 이후(4/22~현재) 실거래 로그 기반 재시뮬레이션
- **최소 샘플 필요:** 각 peak 구간별 20건 이상 (S/A/B급 각각)

### 가설 H2: peak 절대값 도달 시 즉시 부분 매도 추가 (3차 트리거)

#### 가설 내용
현재 SPLIT_1ST(40%) + SPLIT_2ND_TRAIL(60%) 2단계 구조 → **3단계 추가**: peak가 +5% 도달 시 추가 30%를 즉시 매도(시장가), 나머지 30%만 trail_drop 2% 계속 적용

#### 구체적 파라미터 제안
- SPLIT_1ST: armed peak +1.5% 이후 drop 0.45% → 40% 매도 (현행 유지)
- **SPLIT_MID**: peak +5.0% 도달 시 30% 즉시 매도 (신규 추가)
- SPLIT_2ND_TRAIL: 나머지 30% → trail_drop 2.0% 계속 (현행 유지)

#### 근거 (실 데이터 기반)
- peak 5%+ 거래: CHIP(6.74%), KAT(7.31%), PENGU-27(5.11%) 3건
- CHIP 사례: 60% 물량이 peak 190(+6.74%) 대비 drop 2.11% = 184에 청산. 만약 30%를 190에 즉시 매도하면: 추가 수익 = 30% × (190-178)/178 × 250,000 = 약 5,056 KRW 추가 (vs 현행 184 청산 대비 +2,814 KRW 개선)

- PENGU-27 사례: peak 14.4(+5.11%) → 2차 청산 14.1(+2.92%). peak에서 30% 매도 시 +2.19%p × 30% = 약 +1,197 KRW 추가
